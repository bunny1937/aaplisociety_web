# Billing Workflow

## Manual Mode (default)

1. Admin opens Generate Bills page
2. Selects members + billing period
3. **Flow 1 — Generate Bills:**
   - Download Bill Template (pre-filled with charges + previous balances)
   - Adjust charge amounts if needed
   - Upload filled template
   - Validate → ExcelPreviewGrid shows per-cell errors
   - Fix errors, re-upload if needed
   - Click Generate Bills → bills created in DB
4. **Flow 2 — Collect Payments:**
   - Download Payment Template (lists all members with bill amounts)
   - Fill `AmountPaid`, `PaymentMethod`, `PaymentDate` per member
   - Upload filled template
   - Validate → ExcelPreviewGrid shows per-cell errors
   - Continue With Valid Rows → payments posted, receipts generated
5. Repeat next month (previous balances carry forward automatically)

## Automatic Mode

- Configured in Society Config → Bill Generation Mode = Automatic
- System auto-generates bills on `billAutoGenerateDay` (1–5 only)
- Before generation, checks if previous month payment collection was uploaded
- If not uploaded: shows warning dialog with "Upload Sheet First" / "Force Generate Anyway"

## Interest Activation

- **VIEW Mode:** Interest warning shown, no new interest added to bill. Interest carries forward to next cycle.
- **APPLICABLE Mode:** Interest added to bill after `interestAfterDays` from due date.

## Key Routes

| Route | Purpose |
|-------|---------|
| `GET /api/billing/excel-template` | Download pre-filled bill template |
| `POST /api/billing/validate-excel` | Validate uploaded bill template |
| `POST /api/billing/generate-from-excel` | Generate bills from validated Excel |
| `POST /api/billing/upload-payments?action=preview` | Preview payment collection upload |
| `POST /api/billing/upload-payments?action=confirm` | Post payments + generate receipts |
| `GET /api/billing/check-payments-uploaded` | Check if prev month payments uploaded |

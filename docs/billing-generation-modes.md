# Billing Generation Modes

## Configuration

Set in **Society Config → Bill Generation Mode**.

Stored in `society.config.billGenerationMode` (enum: `MANUAL` | `AUTOMATIC`, default: `MANUAL`).

---

## MANUAL Mode (default)

Admin-driven workflow. No automatic generation.

**Steps:**
1. Admin goes to Generate Bills page
2. Selects members and billing period
3. Downloads bill template (pre-filled)
4. Uploads filled template
5. Validates + generates bills

**When to use:** Most societies. Admin has full control over each billing run.

---

## AUTOMATIC Mode

System checks `billAutoGenerateDay` (1–5) and auto-generates on that day.

**Additional config field:** `billAutoGenerateDay` (Number, 1–5, default: 1)

**Pre-generation check:**
Before auto-generating, the system checks if the previous month's payment collection has been uploaded (`paymentUploadedAt` field on Bill records).

- If uploaded → generate normally
- If not uploaded → show warning:
  - "Upload Sheet First" → redirects admin to upload payments for previous period
  - "Force Generate Anyway" → generates without payment upload

**Why 1–5 only:** Early month generation ensures previous month interest is correctly accounted for. Generation after the 5th risks interest calculation drift.

---

## Schema Fields

| Field | Type | Default | Purpose |
|-------|------|---------|---------|
| `billGenerationMode` | `"MANUAL" \| "AUTOMATIC"` | `"MANUAL"` | Generation trigger |
| `billAutoGenerateDay` | Number (1–5) | `1` | Day of month for auto-generation |

---

## Related API

`GET /api/billing/check-payments-uploaded?periodId=2026-04`

Returns `{ uploaded: true/false }` — checks if any Bill in that period has `paymentUploadedAt` set.
